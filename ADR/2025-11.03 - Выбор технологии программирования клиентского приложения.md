
**Дата:** 7 ноября 2025 г.

**Статус:** Принято

**Контекст:** 

Для разработки клиентского приложения системы поиска и резервирования билетов на концерты необходимо выбрать подходящую технологию фронтенд-разработки. От этого выбора зависит производительность интерфейса, скорость разработки, удобство пользовательского взаимодействия и возможность масштабирования приложения в будущем.

**Рассмотренные варианты:**

1. **ReactJS:** 
Плюсы:
- Большое сообщество и активное развитие экосистемы.
- Компонентный подход, упрощающий поддержку и переиспользование кода.
- Высокая производительность за счёт виртуального DOM.
- Гибкость и совместимость с различными инструментами (Redux, React Query, TailwindCSS).

Минусы:
- Нет встроенных механизмов маршрутизации и рендеринга на сервере.
- Требует настройки сборки и инфраструктуры проекта.

2. **Vue.js:** 
Плюсы:
- Простота освоения и читаемость кода.
- Хорошо подходит для быстрого прототипирования.
- Меньше кода для базового функционала.

Минусы:
- Меньше зрелых инструментов для крупных корпоративных приложений.
- Сложнее интеграция с SSR и микросервисной архитектурой.

3. **Angular:** 
Плюсы:
- Полноценный фреймворк с готовыми модулями и строгой структурой.
- Отличная поддержка корпоративных приложений.
- Встроенный модуль для маршрутизации

Минусы:
- Крутая кривая обучения.
- Более тяжеловесные сборки.
- Менее гибкий подход по сравнению с React/Next.js.

5. **Next.js:** 
Плюсы:
- Поддержка SSR (Server-Side Rendering) и SSG (Static Site Generation) для ускорения загрузки и улучшения SEO.
- Простая маршрутизация и оптимизация изображений “из коробки”.
- Интеграция с API и возможность гибкой настройки.
- Хорошо масштабируется и поддерживает микрофронтенд-архитектуру.
- Отличная поддержка TypeScript и инструментов CI/CD.

Минусы:
- Сложнее деплой по сравнению с чистым React.
- Необходимость настройки серверной инфраструктуры для SSR.

**Решение:**

Выбран Next.js

**Обоснование:**

- **Производительность и SEO:** Благодаря серверному рендерингу (SSR) и статической генерации (SSG), Next.js обеспечивает быструю загрузку страниц и высокие показатели SEO, что критично для публичного сервиса бронирования.
- **Масштабируемость:** Архитектура Next.js позволяет безболезненно масштабировать приложение при росте числа пользователей и функциональности.
- **Гибкость и совместимость:** Использование React внутри Next.js предоставляет широкие возможности интеграции с REST и GraphQL API, а также сторонними библиотеками и платежными системами.
- **Экосистема и поддержка:** Активное сообщество, регулярные обновления и поддержка со стороны Vercel гарантируют стабильность и долгосрочную актуальность технологии.
- **Безопасность:** Next.js поддерживает современные механизмы защиты данных, включая настройку заголовков безопасности и HTTPS, что важно для соблюдения требований PCI DSS и защиты пользовательских данных.
- **Мобильная адаптивность:** Приложения на Next.js легко адаптируются под различные устройства и экраны, обеспечивая качественный UX на мобильных, планшетах и десктопах.

**Последствия:**

- Клиентское приложение будет реализовано на Next.js с использованием React и TypeScript.
- Для управления состоянием будут применены Redux Toolkit и React Query.
- Настраивается CI/CD для автоматической сборки и деплоя приложения.
- Разработка и тестирование будут упрощены за счёт единого стека с серверной частью (Node.js / NestJS).
- Возможна дополнительная оптимизация SSR при высоких нагрузках и масштабировании.
